## 3. 关键设计发现（改写 F002 口径，比 spec 预期更好）

1. **★股息率历史 = TR−PR 反推（21.5y）**：csindex 估值接口只回 ~20 天股息率（首轮探针死胡同），但 **H20269 全收益指数存在且 2005 起可得**——滚动 1 年 `TR_return − PR_return` 即指数自身股息率序列，深度 21.5 年，远优于任何直接估值端点。legulegu 市场级股息率（21.5y）作 robustness secondary。
2. **★回测主口径 = H20269 全收益指数**：ETF sina 日线为**未复权价**（不含分红——红利类标的分红是收益主体，直接用会系统性低估）；全收益指数才是防守腿收益的正确度量。分层：**指数 TR 主回测（21.5y，无成本口径标注）+ 512890 sina bars 可实施性/成本层（2019 起，7.46y）**。这与 spec §0"ETF 窗口 + 指数 secondary"相比是口径倒置（指数为主），理由如上，属探针驱动的诚实修正。
3. **窗口红利**：21.5y 覆盖 2008/2015/2018 三轮完整熊市 + 2022-24 调整 + 2025 跑输——风格周期覆盖远超预期（spec 曾担心仅 2018 起一轮）。
4. **死胡同记录**：eastmoney `fund_etf_hist_em` 在 dev box SSL 失败（push2his host，B062 已知 off-box 问题）——弃用；csindex 估值接口深度不足——弃用（仅留作日常新鲜度对照可选项）。

## 4. 诚实边界

- ETF 未复权价不能直接算总收益——F002 必须用 TR 指数为收益主口径，ETF 层只做可交易性/滑点/手数检验；ETF 跟踪误差与费率（~0.5%/年管理+托管）在 F002 成本口径中显式扣减。
- legulegu（乐咕）为第三方聚合站，仅作 secondary，不进关键判定路径。
- TR−PR 反推的是**指数口径股息率**（成分股加权），与 ETF 实际分红时点有差；利差信号用指数口径一致即可（分子分母同为指数/官方口径）。
- 所有序列 staleness=1 天（07-03 收盘数据 07-04 可得）——日刷节奏兼容。
