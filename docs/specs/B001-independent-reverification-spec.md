# B001 Independent Reverification Spec

## Background

B001 produced the initial strategy research documentation set. Its original consistency review and signoff were generated by the same session that produced the artifacts, violating Harness separation of Planner/Generator and Evaluator responsibilities.

The invalidated reports are recorded in `docs/test-reports/harness-violation-self-signoff-2026-05-12.md` and must not be used as formal signoff evidence.

## Goal

Run a Codex-only independent reverification batch for B001.

## Scope

Independent Evaluator must review:

- `docs/specs/B001-strategy-research-roadmap-spec.md`
- `docs/strategy/01-global-etf-momentum-rotation.md`
- `docs/strategy/02-risk-parity-vol-target.md`
- `docs/strategy/03-us-quality-momentum.md`
- `docs/strategy/04-hk-china-etf-small-allocation.md`
- `docs/strategy/05-ai-news-filing-risk-filter.md`
- `docs/test-reports/harness-violation-self-signoff-2026-05-12.md`

## Out Of Scope

- Modifying strategy documents.
- Implementing trading code.
- Running broker or live-money tests.
- Treating invalidated B001 self-signoff artifacts as formal evidence.

## Features

### F001 Independent Review

Executor: codex.

Evaluator must check that the five B001 strategy documents are consistent with B001 spec and with each other on:

- USD 100k-500k personal-account assumption.
- Low-frequency robust strategy style.
- Strategy hierarchy: ETF momentum and risk parity as core, US quality momentum as enhancement, Hong Kong / China ETF as small allocation, AI as risk filter only.
- Capital allocation ranges and exposure limits.
- Data requirements and anti-lookahead expectations.
- Paper Trading gates.
- Live-readiness gates.
- AI usage boundaries.

Acceptance: independent review report is created under `docs/test-reports/` and explicitly states PASS/PARTIAL/FAIL with findings.

### F002 Formal Signoff

Executor: codex.

If F001 passes, Evaluator must create a new independent B001 signoff report under `docs/test-reports/` and update `progress.json.docs.signoff` to that new report.

Acceptance: signoff report exists, `features.json` marks F001/F002 completed, and `progress.json.status` is `done`.

## Safety Rules

- Current Planner must not perform this review or signoff.
- Prior invalidated B001 reports remain invalid even if this reverification passes.
- No real broker, API key, or real-money testing is involved.
