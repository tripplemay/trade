# `data/snapshots/` — real-market-data ingest layout

This directory is the **on-disk staging area** for the Real Data
backfill (B028 onwards). The contents are generated by
`scripts/backfill_prices.py` (B028 F002) and consumed by
`trade/data/loader.py` once B028 F003 wires the unified read path.

The CSV / parquet payloads are too large to commit; only the
directory layout + this README + per-vendor schema samples stay in
git (see `.gitignore`). Re-generate the contents by running the
backfill script from a clean checkout.

## Layout

```
data/snapshots/
├── prices/
│   ├── tiingo/                     # vendor-raw OHLCV (one file per ticker × range)
│   │   ├── SPY-2014-01-01-2026-05-26.csv
│   │   └── ...
│   └── unified/                    # fixture-shaped roll-up
│       └── prices_daily.csv        # every ticker × every date, one file
└── fundamentals/                   # reserved for B029 (SEC EDGAR XBRL)
    ├── sec_edgar/
    └── unified/
```

### Vendor-raw (`prices/tiingo/`)

* One CSV per `(ticker, start_date, end_date)` fetch.
* Naming: `{ticker}-{start_date}-{end_date}.csv` (ISO-8601 dates).
* Schema: identical to the unified table (see below) — kept separate
  so a future cross-check or audit can confirm the vendor's exact
  response without re-fetching.
* Re-running the script for a ticker overwrites the existing vendor
  file (`scripts/backfill_prices.py` uses an atomic `tempfile +
  os.replace`, so a kill mid-write leaves the previous file intact).

### Unified (`prices/unified/prices_daily.csv`)

* Single CSV holding `(date, ticker, open, high, low, close, adj_close, volume)`
  rows for every ticker × date pair the backfill has ever seen.
* Columns + order match the `B025 us_quality_momentum` fixture
  (`data/fixtures/us_quality_momentum/prices_daily.csv`) bit-for-bit
  so `trade/data/loader.py` can swap to the unified path with no
  schema change to downstream strategies.
* Sorted by `(ticker, date)` ascending; rows are deduped by
  `(ticker, date)` so a re-run of an overlapping range produces no
  duplicates.

### Fundamentals (reserved)

Empty until B029 lands the SEC EDGAR PIT XBRL ingest. Layout mirrors
the prices side: `sec_edgar/` for vendor-raw, `unified/` for the
fixture-shaped roll-up.

## How to (re)generate

Prerequisites:

* `TIINGO_API_KEY` exported in the shell (same secret B027 pushes
  to the production VM; see `workbench/backend/.env.example`).
* `scripts/` on the PYTHONPATH (the script self-injects it; running
  with `python scripts/backfill_prices.py` from the repo root is
  enough).

```bash
# Default — Master sleeve + B025 real tickers + ADR proxies (52 tickers).
python scripts/backfill_prices.py \
    --from 2014-01-01 \
    --to 2026-05-26 \
    --universe master

# Or pick a subset.
python scripts/backfill_prices.py \
    --from 2024-01-01 \
    --to 2024-06-30 \
    --tickers SPY,QQQ,IEF
```

The B027 `MonthlyBudgetGuard` is active on every fetch so the script
halts with `BudgetExceeded` rather than overrunning the Tiingo
Starter $10 / month cap. A full universe re-fetch is ≈ 52 Tiingo
requests; at the Starter tier rate limit of 60 req / hour the
script completes in roughly an hour with no rate-limit hits.

## Cross-checking

After a backfill run, validate against yfinance:

```bash
python scripts/validate_snapshot.py --tickers SPY,QQQ,IEF,GLD,TLT
```

Exit code 0 = every sampled `(ticker, date)` pair within 0.5 %;
exit code 1 = at least one discrepancy (table printed for review).
The cross-check report belongs at
`docs/test-reports/B028-cross-check-<YYYY-MM-DD>.md`.

## Why this layout

Per `docs/product/data-source-evaluation-2026-05.md` §7.1, splitting
the vendor-raw and fixture-shaped layers gives us:

* **Provenance.** The raw CSV per ticker is the trail back to the
  vendor's exact response on a given date; a future audit of a
  retroactive adjustment can compare against it.
* **Read-path agility.** The unified file is the only thing the
  strategy code reads. Swapping vendors (B028 ADR proxy / B033+
  alternative source) only requires producing a new unified file —
  the strategy code stays untouched.
* **Storage efficiency.** Vendor-raw files compress well per ticker;
  the unified file fits a `pandas.read_csv` shape downstream code
  already knows.
